How it works

Three data layers, combined — so trees, shadows, and driveways don't fool the number.

1

Find the property

The address resolves to the authoritative parcel boundary — the legal lot lines, not an approximation.

2

Read the ground cover

Current aerial imagery is analyzed to separate turf from house, driveway, and paving.

3

Measure with 3D ground data

LiDAR ground returns confirm genuinely mowable turf — even under tree canopy — and set the square footage. The strip out to the curb is included, because that's what gets mowed.

Why it's accurate

  • Authoritative parcel boundaries, not inferred lot lines.
  • Two independent sensors cover each other's blind spots — imagery can't see lawn under trees; 3D ground data can.
  • The square footage is measured from ground points, so roofs and paving are excluded by nature.
  • Fails clearly when it can't be sure, instead of returning a confident wrong number.

Built like production software

  • Ports-and-adapters architecture — new regions plug in without touching the core.
  • Continuous integration and deployment to a live hosted API.
  • A byte-identical regression gate guarding every change to the math.
  • A fast, fully offline test suite; reproducible, fully pinned builds.
